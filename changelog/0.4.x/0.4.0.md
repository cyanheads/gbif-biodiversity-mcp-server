# 0.4.0 — 2026-06-30

## Added

- **`datasetKey`** input on `gbif_search_occurrences` and `gbif_occurrence_facets` — scopes occurrence retrieval and facet aggregation to a single dataset by its GBIF dataset UUID, mirroring the filter `gbif_count_occurrences` already exposes. Obtain a key from `gbif_search_datasets`, `gbif_get_dataset`, a `DATASET_KEY` facet, or the `datasetKey` field on an occurrence record. ([#25](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/25))

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.10.9` → `^0.10.10`. The lockfile re-resolve also moved transitive `hono` `4.12.26` → `4.12.27` and `@hono/node-server` `2.0.5` → `2.0.6`.

## Removed

- **`publish:compat`** npm script — the `@cyanheads/gbif-mcp-server` compat-alias publish path.

## Fixed

- **`hasCoordinate`** description on `gbif_search_occurrences` corrected: `false` returns ONLY records without coordinates (the prior text claimed it would *include* such records); omit the parameter to return all records regardless of coordinate presence. The same correction was applied to `docs/design.md`. ([#24](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/24))

## Security

- **Transitive `js-yaml`** `3.14.2` → `3.15.0`, pulled in by the mcp-ts-core `^0.10.10` lockfile re-resolve, clears [GHSA-h67p-54hq-rp68](https://github.com/nodeca/js-yaml/security/advisories/GHSA-h67p-54hq-rp68) (CVE-2026-53550) — a moderate quadratic-complexity DoS in YAML merge-key handling. `bun audit` reports no vulnerabilities.
