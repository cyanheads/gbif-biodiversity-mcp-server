# 0.5.2 — 2026-07-13

## Added

- **`gbif_occurrence_facets`** — new `facetOffset` input (default 0) threads through to GBIF's `facetOffset`, and the enrichment now echoes the applied `facetOffset` plus a `moreValuesLikely` heuristic flag (true when a full `facetLimit` page came back). High-cardinality dimensions like `DATASET_KEY` are now pageable past the first `facetLimit` values — advance `facetOffset` by `facetLimit` for the next ranked page. ([#32](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/32))

## Fixed

- **`gbif_search_datasets`** — each result whose description was shortened to the 300-character preview now carries `descriptionTruncated: true` in `structuredContent`, so structured-only clients can tell the text is partial and fetch the full description via `gbif_get_dataset`. Previously only the rendered `content[]` showed the ellipsis. ([#30](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/30))
- **`gbif_get_species_classification`** — the tool and output-field descriptions now consistently state the chain runs root-to-immediate-parent and excludes the queried taxon (call `gbif_get_species` for its own record), resolving the contradiction between the top-level description ("down to the taxon itself"), the field description, and the live behavior. ([#29](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/29))
