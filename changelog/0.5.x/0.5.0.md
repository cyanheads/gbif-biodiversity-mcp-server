# 0.5.0 — 2026-06-30

## Added

- **`gbif_bulk_match_species`** — resolve up to 50 scientific names to GBIF backbone taxon keys in one call, the batch counterpart to `gbif_match_species` for checklist, inventory, and species-list workflows that would otherwise need one round trip per name. Returns one entry per input name, in input order; an unmatched name yields `matchType NONE` and a per-name lookup failure yields `matchType ERROR`, neither sinking the rest of the batch. Reuses the existing `matchSpecies` service over `Promise.all` — GBIF has no native bulk match endpoint. Brings the tool count to 13. ([#16](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/16))

## Fixed

- **`gbif_get_species_children`** no longer reports a fabricated `totalCount: 0` — the GBIF `/species/{key}/children` endpoint carries no count field, so the value was always `0`. The field is removed; genuine page truncation is now disclosed via new `truncated`, `shown`, and `cap` enrichment fields derived from the upstream `endOfRecords` flag. ([#3](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/3))
- **`matchType`** description on `gbif_match_species` corrected — `HIGHERORDER` → `HIGHERRANK`, the value GBIF actually returns for a higher-rank match.
