# 0.5.1 — 2026-07-13

## Added

- **`gbif_get_occurrence`** — now returns `occurrenceID`, full `class`/`classKey`, GADM administrative geography (levels 0–2, each with a stable `gid` and `name`), and source `identifiers`, closing the gap between the advertised output schema and what the handler actually returned. ([#27](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/27))
- **`gbif_get_dataset`** and the **`gbif://dataset/{datasetKey}`** resource — now return `temporalCoverages` and `geographicCoverages`; the resource additionally gains `contacts`/`contactsTotal`/`contactsReturned` (capped at 10), matching the tool's existing contact projection. Both share the new `projectContacts`/`compactTemporalCoverages`/`compactGeographicCoverages` helpers in `tools/utils.ts`. ([#28](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/28))

## Fixed

- **`gbif_get_species`**, **`gbif_search_species`**, and the **`gbif://species/{taxonKey}`** resource — read `raw.class` instead of the always-`undefined` `raw.clazz`, which had silently dropped the classification's class name from every response. ([#34](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/34))
- **`gbif_get_species`** classification text — every rank from kingdom through species now renders a key-only fallback (e.g. `Class key: 359`) when GBIF supplies a key but no name, instead of silently dropping the rank from the rendered text. ([#31](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/31))

## Changed

- **Supply-chain hardening** — `bunfig.toml` adds the Socket install scanner (`@socketsecurity/bun-security-scanner`) and a 3-day `minimumReleaseAge` gate on new package versions (framework releases excluded — adopted same-day); `Dockerfile` installs with `--ignore-scripts` plus BuildKit cache mounts; `devcheck`'s outdated-package check now recognizes rows held back by `minimumReleaseAge` instead of flagging them as unexpected; `typescript` added to the outdated allowlist (held at `^6`); new `.github/SECURITY.md` documents the vulnerability-reporting process.

## Dependencies
