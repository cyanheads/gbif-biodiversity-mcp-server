# 0.2.1 — 2026-05-24

## Added

- **`gbif_occurrence_facets`** — `STATE_PROVINCE` added to the `facet` enum, enabling state/province-level occurrence breakdowns within a country ([#11](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/11)).
- **`gbif_search_occurrences`** — new `coordinateUncertaintyInMeters` parameter (range format `"min,max"`, e.g. `"0,1000"`) filters records by coordinate precision; passed through to the GBIF API ([#12](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/12)).

## Changed

- **`gbif_occurrence_facets` `format()`** — percentage now uses `totalOccurrences` as the denominator (was `maxCount`, the top bin count). Label updated from "% of top" to "% of total" ([#10](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/10)).

## Fixed

- **`year` parameter descriptions** — appended "Both endpoints inclusive." to `gbif_search_occurrences`, `gbif_count_occurrences`, and `gbif_occurrence_facets` ([#13](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/13)).
- **`taxonKey` parameter descriptions** — appended "Matches the given taxon and all descendant taxa (subspecies, varieties, etc.)" to `gbif_search_occurrences`, `gbif_count_occurrences`, and `gbif_occurrence_facets` ([#14](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/14)).
- **`isInCluster` parameter description** — updated to explain that GBIF only exposes a membership boolean, not a cluster identifier; documents `isInCluster: false` as the de-duplication path ([#15](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/15)).
