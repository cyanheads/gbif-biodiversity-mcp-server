# 0.2.4 — 2026-05-30

## Changed

- **`gbif_search_occurrences`** — pagination metadata (`totalCount`, `offset`, `limit`, `endOfRecords`) moved from `output` to `enrichment`; both channels carry the same data. `output` now contains only `occurrences`.
- **`gbif_search_species`** — pagination metadata (`totalCount`, `offset`, `limit`, `endOfRecords`) moved to `enrichment`. `output` now contains only `taxa`.
- **`gbif_search_datasets`** — pagination metadata (`totalCount`, `offset`, `limit`, `endOfRecords`) moved to `enrichment`. `output` now contains only `datasets`.
- **`gbif_search_publishers`** — pagination metadata (`totalCount`, `offset`, `limit`, `endOfRecords`) moved to `enrichment`. `output` now contains only `publishers`.
- **`gbif_get_species_children`** — pagination metadata (`offset`, `limit`, `endOfRecords`) moved to `enrichment`. `output` now contains only `children`.
- **`gbif_occurrence_facets`** — `facetLimit` added to `enrichment` (was not previously surfaced).

## Added

- **`enrichment.notice`** — optional guidance field on all six affected tools. Populated when results are empty (zero-hit query guidance) or when pagination offset overshoots `totalCount` (reset instruction). Absent on successful result pages. Reaches both `structuredContent` and `content[]`.
- **`@cyanheads/mcp-ts-core`** `^0.9.13 → ^0.9.16` — framework update enabling the `enrichment` API.
