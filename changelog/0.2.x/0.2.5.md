# 0.2.5 — 2026-06-02

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.16 → ^0.9.21` — framework update; notable changes:
  - HTTP transport per-request log context fix: per-request logs and traces now carry fresh request + trace/span IDs instead of the frozen boot context.
  - `fetchWithTimeout` strips query-string secrets (e.g. `?api_key=`) from error messages and logs.
  - `withRetry` fails fast on non-retryable errors; `ctx.fail` auto-populates the `retryable` flag.
- **`vitest`** `^4.1.7 → ^4.1.8` — in-range dev-dep update.
- **Client config key** in README install snippets renamed from `gbif` to `gbif-biodiversity-mcp-server` (full package name convention).
- **Plugin manifests** (`.claude-plugin/`, `.codex-plugin/`) — `mcpServers` args simplified to match current install patterns.
- **`scripts/devcheck.ts`** — two new checks from framework sync: Open-Indexed Interfaces (existence-guarded, skipped in consumer projects) and Skill Versions (warns when skill body changes without a version bump).
- **`scripts/`** — `release-github.ts` and `check-skill-versions.ts` added from framework template sync.
- **`skills/`** — 8 skills updated, `api-mirror` and `orchestrations` added from framework 0.9.21 template sync.
