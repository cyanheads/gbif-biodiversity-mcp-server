# 0.2.7 — 2026-06-12

## Added

- **`createApp({ name, title })`** — explicit server identity in `src/index.ts`, both set to the unscoped package name `gbif-biodiversity-mcp-server` so the served `server_name` and `/.well-known/mcp.json` carry the machine name rather than the npm scope.
- **`totalCount`** in `gbif_get_species_children` enrichment — total direct children before pagination, sourced from the upstream `count`. Satisfies the framework's `capped-list-no-truncation` lint rule.
- **`scripts/clean-mcpb.ts`** — post-pack `.mcpb` cleaner wired into the `bundle` script: runs `mcpb clean` (dev-dependency prune) then strips dependency-shipped agent docs (`node_modules/**` `skills/`, `.claude/`, `.agents/`, `SKILL.md`) that root-anchored `.mcpbignore` patterns cannot reach.
- **`HEALTHCHECK`** in the `Dockerfile` — bun-native `fetch` against `/healthz` (the slim image ships no curl/wget). `org.opencontainers.image.version` label stamped from the `APP_VERSION` build arg.

## Changed

- **`scripts/lint-packaging.ts`** — added bundle-content guards (dev-dir exclusion, unanchored-pattern detection, critical-runtime-path protection), a post-bundle content check that fails on `node_modules/**` agent-doc entries, and an entrypoint/manifest identity check that pins `name`/`title`/`display_name` to the unscoped package name.
- **`scripts/check-framework-antipatterns.ts`** — added the `coerce-boolean-env-flag` rule (`z.coerce.boolean()` on an env flag can't be disabled via the environment — use `z.stringbool()`); comment lines are now skipped so documentation mentions don't false-positive.
- **`.mcpbignore`** — root dev-dir patterns anchored with a leading `/` so they no longer strip nested runtime paths under `node_modules/`.
- **Vendored skills** re-synced to mcp-ts-core 0.10.6; added the `techniques` skill (outline-on-overflow reference).
