# 0.1.6 — 2026-05-24

## Fixed

- **`gbif_get_species`, `gbif_get_occurrence`, `gbif_get_dataset`** — 404 responses from GBIF were surfaced as unhandled `McpError(-32001)` instead of the tool's `not_found` error contract ([#6](https://github.com/cyanheads/gbif-mcp-server/issues/6)).
- **`gbif_get_species_classification`** — nonexistent taxon keys returned a silent empty classification instead of `not_found`; added existence validation via `getSpecies` when the parents array is empty ([#7](https://github.com/cyanheads/gbif-mcp-server/issues/7)).
- **`gbif_search_occurrences`** — requests exceeding the GBIF ~100,000 `offset+limit` cap triggered 4 upstream retries before failing; now throws `upstream_error` pre-flight before the network call ([#8](https://github.com/cyanheads/gbif-mcp-server/issues/8)).
- **`gbif_match_species`** — description and `name` parameter docstring implied common-name support; corrected to scientific names only with a pointer to `gbif_search_species` for vernacular searches ([#9](https://github.com/cyanheads/gbif-mcp-server/issues/9)).
