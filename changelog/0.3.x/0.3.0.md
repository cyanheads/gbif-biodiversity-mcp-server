# 0.3.0 — 2026-06-30

## Added

- **`contactLimit`** input on `gbif_get_dataset` (default 10, max 100; `0` suppresses contact detail while still reporting the count), plus `contactsTotal` and `contactsReturned` output fields. Caps the contact list so high-contact datasets like eBird no longer dominate the response, while the full count stays visible. ([#26](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/26))

## Fixed

- **`gbif://species/{taxonKey}` and `gbif://dataset/{datasetKey}` resource reads** now map a missing record to a clean domain `not_found` error with recovery hints, instead of leaking the raw upstream GBIF 404 envelope — matching the behavior of the equivalent tools. ([#22](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/22))
- **HTML tags stripped** from `gbif_get_species` `publishedIn` and the `gbif://dataset/{datasetKey}` `description`, so citations and abstracts reach clients as plain text. ([#23](https://github.com/cyanheads/gbif-biodiversity-mcp-server/issues/23))
